## 第 1 步：准备 GitHub 仓库

### 1.1 在 GitHub 创建新仓库

1. 打开浏览器，访问：https://github.com/new
2. 填写：
   - **Repository name**: `dogdoggo`
   - 选择 **Public**（公开，hackathon 评委可以看）
   - **不要勾选**任何复选框
3. 点击 **"Create repository"**

### 1.2 推送代码（在终端执行）

```bash
# 进入项目目录
cd /Users/ander/Library/CloudStorage/OneDrive-UAM/桌面/cursor_h

# 初始化 git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "feat: DogDogGo Next.js listo para Vercel"

# 连接到 GitHub（把 YOUR_USERNAME 改成你的 GitHub 用户名！）
git remote add origin https://github.com/YOUR_USERNAME/dogdoggo.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

**重要**：记得替换 `YOUR_USERNAME`！

✅ 完成！刷新 GitHub 页面，你应该看到代码了。

---

## 第 2 步：在 Vercel 导入并部署

### 2.1 打开 Vercel

浏览器访问：https://vercel.com

### 2.2 登录

- 如果没有账号：点击 **"Sign Up"**，选择 **"Continue with GitHub"**
- 如果有账号：点击 **"Log In"**，选择 **"Continue with GitHub"**

### 2.3 授权 GitHub

Vercel 会要求访问你的 GitHub，点击 **"Authorize Vercel"**

### 2.4 导入项目

1. 点击右上角 **"Add New..."** → **"Project"**
2. 你会看到你的 GitHub 仓库列表
3. 找到 `dogdoggo`
4. 点击旁边的 **"Import"** 按钮

### 2.5 配置项目

**Configure Project** 页面：

**Project Name**: `dogdoggo`（保持默认）

**Framework Preset**: Next.js ✅ （自动检测）

**Root Directory**: `./` ✅（保持默认）

**Build and Output Settings**: 全部保持默认 ✅

**Environment Variables** - 展开这个部分：

添加一个变量：

```
Name: NEXTAUTH_SECRET
Value: [在终端运行 openssl rand -base64 32，复制输出]
```

**如何获取 Value**：

打开你的终端，运行：
```bash
openssl rand -base64 32
```

会输出类似：`kJ3mP9xR2vN8qT5wY1zC7bF4gH6jL0sA`

复制这个，粘贴到 Value 框中。

### 2.6 开始部署

点击蓝色大按钮 **"Deploy"**

等待 2-3 分钟... Vercel 正在：
- 📦 安装依赖
- 🔨 构建项目  
- 🚀 部署到全球 CDN

你会看到进度条和日志。

## 第 3 步：添加数据库（最重要！）

### 3.1 等待部署完成

当看到：
```
✓ Build Completed
✓ Deployment Ready
🎉 Congratulations!
```

### 3.2 不要访问应用！先添加数据库

在 Vercel 部署成功页面：

1. 点击顶部导航栏的 **"Storage"** 标签
2. 点击 **"Create Database"** 按钮
3. 选择 **"Postgres"** 卡片
4. Database Name: 保持默认即可
5. Region: 选择 **"Frankfurt, Germany (fra1)"** 或 **"Dublin, Ireland (dub1)"**（离马德里近）
6. 点击 **"Create"** 按钮

等待 30-60 秒... 数据库正在创建

### 3.3 连接数据库到项目

创建完成后，Vercel 会问：

**"Connect Database to Project"**

1. 你会看到你的项目名 **"dogdoggo"** 
2. 确保它被选中（勾选框）
3. 点击 **"Connect"** 按钮

✅ **环境变量自动配置完成！**

你会看到类似这些变量被添加：
- `POSTGRES_URL`
- `POSTGRES_PRISMA_URL`
- `POSTGRES_URL_NON_POOLING`
- 等等...

---

## 第 4 步：重新部署（让数据库生效）

### 4.1 触发重新部署

因为数据库是在第一次部署后添加的，需要重新部署来创建表：

1. 点击顶部的 **"Deployments"** 标签
2. 看到最新的 deployment（最上面那个，有绿色勾号）
3. 点击右边的三个点图标 **"..."**
4. 选择 **"Redeploy"**
5. 弹窗中，点击 **"Redeploy"** 确认

等待 1-2 分钟...

## 🐛 如果遇到问题

### 问题：部署失败

**检查**:
1. Vercel Dashboard → Deployments → 点击失败的 → Build Logs
2. 看红色错误信息
3. 通常是缺少环境变量

**解决**:
```bash
# 确保 NEXTAUTH_SECRET 已添加
# Settings → Environment Variables
```

### 问题：应用打开显示 error

**检查**:
1. 数据库是否已创建并连接
2. 是否重新部署了（在添加数据库后）

**解决**:
- Deployments → Redeploy

### 问题：可以访问但不能注册

**检查**:
1. Storage → Postgres → Data
2. 查看是否有表

**解决**:
```bash
# 本地连接生产数据库
vercel env pull .env.production
npx prisma db push
```
